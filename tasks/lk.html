Постановка задачи: Личный кабинет пользователя

Реализовать функционал страниц Личного кабинета пользователя. Пользователь должен иметь понятный доступ к своим данным, активности и участию в конкурсах через единый интерфейс вкладок.

Кликабельный прототип

https://28c197fc.pp-5vu.pages.dev/

1. Общий пользовательский сценарий

  1. Пользователь авторизуется и попадает в раздел «Личный кабинет».
  2. Внутри ЛК пользователь видит 4 вкладки:
    • Профиль
    • Диктанты
    • Опросы
    • Конкурсы
  3. Пользователь может свободно переключаться между вкладками.
  4. На всех страницах ЛК сохраняется единый стиль, хлебные крошки и заголовок раздела.

1.1. Пользовательские уведомления в шапке

Назначение: быстрые уведомления о статусах заявок в конкурсах.

Требования:

  • Блок уведомлений доступен только авторизованному пользователю (иконка «колокольчик» в шапке рядом с профилем).
  • Если есть непрочитанные уведомления — на колокольчике отображается индикатор (точка).
  • По клику на колокольчик открывается dropdown со списком уведомлений.
  • Каждое уведомление отображает:
    • Заголовок (например: «Ваша заявка одобрена», «Ваша заявка победила в конкурсе»).
    • Дата и время создания (формат: ДД.ММ.ГГГГ, ЧЧ:ММ).
    • Название конкурса — это ссылка на страницу конкурса по пути /conest/:contestId.
    • Заявка (название заявки) и Номинация.
  • Клик по уведомлению помечает его как прочитанное (в том числе клик по названию конкурса).
  • Прочитанные уведомления имеют отметку «Прочитано».

2. Структура раздела ЛК

2.1. Вкладка «Профиль»

Назначение: хранение и редактирование персональных данных пользователя.

На странице выводятся поля профиля:

  1. Регион
  2. Адрес
  3. Фамилия
  4. Имя
  5. Отчество
  6. Пол
  7. Дата рождения
  8. Email
  9. Являюсь

Действия пользователя:

  • изменить данные в полях;
  • нажать кнопку «Сохранить»;
  • получить подтверждение об успешном сохранении или сообщение об ошибке.

2.2. Вкладка «Диктанты»

Назначение: просмотр результатов пройденных диктантов.

Для каждого результата отображается:

  1. Название диктанта
  2. Дата прохождения
  3. Результат (например: «X из 30 (Y%)»)
  4. Миниатюра сертификата (если есть)
  5. Кнопка/действие «Скачать сертификат»

Пустое состояние:

  • если результатов нет, выводится понятное сообщение: пользователь пока не проходил диктанты.

2.3. Вкладка «Опросы»

Назначение: просмотр истории участия в опросах.

Для каждого опроса отображается:

  1. Название опроса
  2. Дата прохождения
  3. Статус/результат (если предусмотрено бизнес-логикой)

Пустое состояние:

  • если пользователь еще не проходил опросы, показывается сообщение о пустой истории.

2.4. Вкладка «Конкурсы»

Назначение: просмотр активности пользователя в конкурсах.

Для каждого конкурса на странице отображается:

  1. Название конкурса
  2. Название проекта
  3. Дата/период проведения
  4. Блок «Моя заявка» (если заявка есть)
  5. Блок «Мои голоса» (если пользователь голосовал)

Доступные переходы:

  • в отдельную страницу конкурса (вне ЛК) по пути /conest/:contestId;
  • в страницу «Моя заявка» по конкретному конкурсу.

3. Страница «Моя заявка»

Страница открывается по маршруту заявки конкретного конкурса.

3.1. Верхняя часть страницы

  1. Заголовок заявки
  2. Статус заявки (например: «На модерации», «Победитель»)
  3. Дата подачи
  4. Строка: «Название конкурса / Название проекта»

Требование к ссылкам:

  • название конкурса — ссылка на вкладку «Конкурсы»;
  • название проекта — ссылка на страницу проекта.

3.2. Блок медиа

  1. Слева отображается основное медиа (фото или видео).
  2. Ниже — миниатюры галереи для переключения медиа.
  3. Если в галерее есть видео, оно показывается первым.
  4. Клик по миниатюре переключает медиа в основном окне.
  5. Клик по большому фото открывает его в увеличенном виде (модальное окно).

3.3. Информация справа от медиа

В правой части блока отображаются:

  1. Номинация (чип)
  2. Регион (чип с иконкой)
  3. Муниципалитет (чип с иконкой)
  4. Количество голосов
  5. Данные заявки

3.4. Дополнительные секции ниже

  1. Эссе — файл, который можно скачать.
  2. Ссылки СМИ — список внешних ссылок на публикации.

4. Навигация и крошки

4.1. Общие требования

  • Для страниц ЛК крошки должны отражать путь пользователя внутри раздела.
  • Для страницы заявки — цепочка: Главная → Личный кабинет → Конкурсы → Моя заявка на конкурс [Название конкурса].

5. Состояния интерфейса

5.1. Загрузка

  • Пока данные не получены, показывать загрузочное состояние (скелетон/заглушка).

5.2. Пустые данные

  • Если данных нет (опросы, диктанты, заявка, ссылки СМИ), показывать аккуратное пустое состояние.

5.3. Ошибка

  • При ошибке загрузки показывать блок с текстом ошибки и кнопкой «Повторить».

6. Критерии приёмки

  • Пользователь видит в ЛК 4 вкладки: Профиль, Диктанты, Опросы, Конкурсы.
  • В профиле отображаются и сохраняются поля пользователя.
  • Во вкладке «Диктанты» отображаются результаты и скачивание сертификатов.
  • Во вкладке «Опросы» отображается история прохождения.
  • Во вкладке «Конкурсы» отображаются конкурсы пользователя, его заявка и его голоса.
  • На странице «Моя заявка» соблюдена структура:
    • заголовок, статус, дата подачи, конкурс/проект;
    • галерея с видео первым;
    • переключение медиа миниатюрами;
    • увеличение фото по клику на большое изображение;
    • чипы номинации/региона/муниципалитета;
    • данные заявки (год, этажи, квартиры);
    • эссе (скачивание), ссылки СМИ.
